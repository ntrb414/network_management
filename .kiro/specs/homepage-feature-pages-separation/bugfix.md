# 首页与功能页面分离 - Bugfix需求文档

## 介绍

当前系统的功能页面（设备管理、配置管理、监控等）错误地出现在Django admin路由下，而不是在独立的首页中。这导致用户体验混乱，功能页面与管理后台混淆。本修复的目标是创建一个独立的首页，展示所有功能模块的按钮，并将功能页面从admin路由下分离出来，使用户权限仅用于内容访问控制，不影响页面位置。

## Bug分析

### 当前行为（缺陷）

1.1 WHEN 用户访问系统时 THEN 系统没有首页，用户无法看到功能模块的统一入口

1.2 WHEN 用户尝试访问功能页面时 THEN 系统将功能页面路由到Django admin路由下（如 `/admin/devices/`），导致功能页面与管理后台混淆

1.3 WHEN 用户权限检查失败时 THEN 系统弹窗显示"无权限"提示，但用户无法从首页清晰地了解自己可以访问哪些功能

1.4 WHEN 用户访问功能页面时 THEN 功能页面在Django admin的侧边栏菜单中注册，导致功能模块与系统管理混淆

### 期望行为（正确）

2.1 WHEN 用户访问系统根路径 `/` 时 THEN 系统应该显示首页，展示所有8个功能模块的按钮（设备管理、配置管理、监控、告警、拓扑、日志、备份、账户管理）

2.2 WHEN 用户点击首页上的功能按钮时 THEN 系统应该导航到对应的功能页面，URL应该在首页路由下（如 `/devices/`、`/configs/` 等），而不是在admin路由下

2.3 WHEN 用户权限检查失败时 THEN 系统应该弹窗显示"无权限"提示，用户需要向管理员申请权限，但首页本身应该始终可访问

2.4 WHEN 用户访问功能页面时 THEN 功能页面不应该在Django admin的侧边栏菜单中注册，功能模块应该与系统管理完全分离

2.5 WHEN 用户具有特定功能的权限时 THEN 系统应该允许用户访问该功能页面并查看相关内容

2.6 WHEN 用户不具有特定功能的权限时 THEN 系统应该在用户尝试访问该功能时弹窗显示"无权限"提示，用户可以返回首页

### 不变行为（防止回归）

3.1 WHEN 用户具有管理员角色时 THEN 系统应该继续允许管理员访问Django admin后台进行系统管理

3.2 WHEN 用户具有普通用户或只读用户角色时 THEN 系统应该继续根据用户权限限制内容查看权限，不影响首页和功能页面的访问

3.3 WHEN 用户在功能页面中进行操作时 THEN 系统应该继续根据用户权限限制操作权限（如只读用户无法修改配置）

3.4 WHEN 系统处理多个并发用户请求时 THEN 系统应该继续正确处理400台设备的数据，支持4个并发用户，不因为首页的添加而降低性能

3.5 WHEN 用户使用不同的浏览器或设备访问系统时 THEN 系统应该继续在所有设备上正确显示功能页面和权限提示
